# Assistive Commons Watch — Claude Context

See [`assistive-commons-watch.md`](assistive-commons-watch.md) for the full project brief: concept, data model, health-scoring formula, UX design decisions, and seed-project list.

## Repo layout

```
_data/projects/       one YAML file per project (source of truth for all project data)
_fetchers/            Python: Pydantic models, GitHub REST fetcher, health-scoring engine
scripts/              CLI tools
  validate_yaml.py    validates all YAMLs against the Pydantic model
  export_schema.py    regenerates schema.json (do not hand-edit schema.json)
  build_json.py       reads YAMLs → writes site/_data/acw.json for the 11ty frontend
design/landscape/     Claude Design handoff for the Landscape homepage
  README.md           pixel-accurate implementation spec (colors, type, layout, behavior)
  Landscape.dc.html   interactive hi-fi prototype (open in browser alongside support.js)
site/                 11ty frontend source (Eleventy input dir)
  _data/acw.json      generated by scripts/build_json.py — do not edit by hand
  _includes/          Nunjucks layout templates
  index.njk           Landscape homepage (three-tab dashboard)
_site/                Eleventy build output (gitignored)
schema.json           CI artifact — do not edit by hand
```

## Development workflow

```bash
# Python setup
pip install -e .

# Node setup (11ty)
npm install

# Build site data from YAMLs (required before running 11ty)
python scripts/build_json.py

# Serve locally
npm run serve       # http://localhost:8080

# Production build
npm run build       # → _site/
```

## Key conventions

- **YAMLs are the source of truth.** The Pydantic model in `_fetchers/models.py` is authoritative. Validate with `python scripts/validate_yaml.py`.
- **`health_tier` / `health_score` in YAML** are written by the nightly fetch workflow after live data is fetched. Manually set values are overwritten on next run.
- **`site/_data/acw.json`** is generated by `scripts/build_json.py` — it bakes in health tiers, area groupings, stats, and sidebar data. 11ty reads it; do not edit it directly.
- **`schema.json`** is generated by `scripts/export_schema.py` — do not edit it directly.
- **`disability_area` ordering matters** — the first area in the list is used as the primary display area on the Landscape page. Eye-gaze projects list `eyegaze` first.

## Design reference

The Landscape homepage is implemented in `site/index.njk` following the spec in `design/landscape/README.md`. That README is the authoritative source for colors, typography, spacing, and interactions. When making frontend changes, check the spec there first.

## Health-scoring summary

```
score = 0.25×activity + 0.30×replicability + 0.15×community + 0.20×at_specific + 0.10×provenance
```

Tier rules (first match wins): Archived → Thriving (≥7.5) → Stable (≥5.5) → Dormant (activity≤2 AND replicability≥5) → At Risk.
